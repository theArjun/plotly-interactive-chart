# Plotly Interactive Chart: Python to Frontend

Build interactive charts in Python with Plotly, export them as JSON (`fig.to_json()`), and render them in the browser using Plotly.js — with runtime controls for theming, hover modes, line styles, marker sizing, legend placement, zoom, and PNG/SVG export — no Python server required at runtime.

**[Live Demo](https://adhikariarjun.com.np/plotly-interactive-chart/)** | **[Source Code](https://github.com/theArjun/plotly-interactive-chart)**

### Features

- Dark / light theme toggle
- Hover mode switching (X Unified, Closest, X, Y, Off)
- Drag mode selection (Zoom, Pan, Select, Lasso)
- Line style control (Solid, Dot, Dash, Dash-Dot)
- Marker size slider
- Legend repositioning (Default, Bottom, Top Left, Hidden)
- One-click zoom reset
- PNG and SVG chart export (2x resolution)
- Scroll-zoom support
- Fully responsive layout

## How It Works

### 1. Build the chart in Python

Use `plotly.express` (or `plotly.graph_objects`) to create a figure from your data:

```python
import plotly.express as px
import polars as pl

df = pl.read_csv("sample_sales.csv", try_parse_dates=True)

fig = px.line(
    df.to_pandas(),
    x="date",
    y="sales",
    color="category",
    markers=True,
    title="Sample Sales Over Time by Category",
)
fig.update_layout(template="plotly_white")
```

### 2. Export the chart to JSON

A Plotly figure serializes to a JSON object with two keys — `data` (the traces) and `layout` (axes, title, theme, etc.). Call `to_json()` and write it to a file:

```python
from pathlib import Path

Path("sample_chart.json").write_text(fig.to_json(), encoding="utf-8")
```

The resulting JSON is a self-contained description of the chart — no Python dependencies needed to render it.

### 3. Render in the browser with Plotly.js

Load the Plotly.js CDN and feed it the JSON:

```html
<script src="https://cdn.plot.ly/plotly-2.35.2.min.js"></script>

<div id="chart"></div>

<script>
  fetch('./sample_chart.json')
    .then(r => r.json())
    .then(fig => Plotly.newPlot('chart', fig.data, fig.layout, { responsive: true }));
</script>
```

`Plotly.newPlot` accepts the same `data` and `layout` objects that Python's `fig.to_json()` produces, so the chart renders identically in the browser — with full interactivity (hover, zoom, pan, legend toggle).

### 4. Runtime controls with Plotly.js API

The included `index.html` goes beyond basic rendering — it uses the Plotly.js API to let users tweak the chart at runtime without reloading:

| Control | Plotly.js API | What it does |
|---|---|---|
| Dark / Light theme | `Plotly.relayout()` | Switches chart colors (grid, font, axes) + page CSS variables |
| Hover mode | `Plotly.relayout({ hovermode })` | X Unified, Closest, X, Y, or Off |
| Drag mode | `Plotly.relayout({ dragmode })` | Zoom, Pan, Select, or Lasso |
| Line style | `Plotly.restyle({ 'line.dash' })` | Solid, Dot, Dash, Dash-Dot |
| Marker size | `Plotly.restyle({ 'marker.size' })` | Range slider (4–20 px) |
| Legend position | `Plotly.relayout()` | Default, Bottom (horizontal), Top Left, Hidden |
| Reset Zoom | `Plotly.relayout({ 'xaxis.autorange': true })` | Restores auto-range on both axes |
| Export PNG / SVG | `Plotly.downloadImage()` | Downloads at 2x scale (1200 x 700) |

The built-in Plotly mode bar is hidden (`displayModeBar: false`) since the custom controls replace it. Scroll-zoom is enabled via config.

## Project Structure

```
main.py              # Reads CSV, builds Plotly figure, exports JSON
sample_sales.csv     # Sample dataset
sample_chart.json    # Exported chart (generated by main.py)
index.html           # Renders the chart using Plotly.js
pyproject.toml       # Python dependencies (plotly, polars, pandas, pyarrow)
```

## Quick Start

```bash
# Install dependencies
uv sync

# Generate the chart JSON
uv run main.py

# Serve the HTML (any static server works)
python -m http.server
# Open http://localhost:8000
```

## Why This Approach?

- **Decoupled frontend/backend** — the Python script is a build step, not a runtime dependency. The frontend is plain HTML + JS.
- **Portable** — the JSON file can be served from a CDN, embedded in a static site, or loaded by any Plotly.js client (React, Vue, etc.).
- **Full fidelity** — the JSON captures the complete figure specification, so the browser chart matches what you see in Python.
